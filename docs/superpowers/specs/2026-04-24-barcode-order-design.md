# バーコード発注リストアプリ 設計書

- 作成日: 2026-04-24
- 対象ディレクトリ: `/Users/yo-chan/アプリ/barcode-order/`

## 1. 概要

スマホのカメラでバーコード（JAN/EAN）を読み取り、品名・数量・発注先を管理できる Web アプリ。最終的に発注リストを CSV として出力できる。

- 想定ユーザー: 診療所の発注担当者
- 発注先: アスクル / カウネット / Amazon / 診療（4択）
- オフライン・ローカル完結（サーバー不要、データはブラウザに保存）

## 2. 要件

### 機能要件

1. スマホカメラで JAN/EAN-13 バーコードを読み取れる
2. 読み取った商品の品名・発注先をブラウザに保存し、次回以降は自動で引き当てる（商品マスタ方式）
3. 同じ商品を再度スキャンすると、発注リストの数量が +1 される
4. 発注リスト上で数量・発注先を手動で変更できる
5. 発注先は アスクル / カウネット / Amazon / 診療 の 4 択
6. 発注リストを CSV（1 ファイル、全発注先混在）でダウンロードできる
7. 商品マスタを一覧・編集・削除できる管理画面がある
8. カメラが使えない環境向けに JAN コードの手動入力フォールバックがある

### 非機能要件

- データはすべて `localStorage` に保存（サーバー不要）
- スマホ（iPhone Safari / Android Chrome）で動作
- ビルド不要でそのまま動く構成
- `https` または `localhost` 配信（カメラ API 要件）

## 3. アーキテクチャ

### 3.1 ファイル構成

```
barcode-order/
├── index.html          # メイン画面（スキャン + 発注リスト + 操作）
├── app.js              # アプリロジック
├── styles.css          # スタイル
├── manifest.json       # PWA 設定（ホーム画面追加用）
├── README.md           # 使い方
└── docs/
    └── superpowers/
        └── specs/
            └── 2026-04-24-barcode-order-design.md
```

- `html5-qrcode` ライブラリは CDN から読み込み（例: `https://unpkg.com/html5-qrcode`）
- ビルド不要。`index.html` を静的配信すれば動作
- 開発時は `python3 -m http.server` 等で `localhost` 配信し、スマホから同一 LAN 経由でアクセス（ただし iOS Safari はカメラ許可のため `https` が必要なケースあり。デプロイ方法は実装時に詰める）

### 3.2 画面構成

単一ページに 3 エリアを縦並びで配置。

1. **スキャンエリア（上部）**
   - 「スキャン開始」ボタン → `html5-qrcode` でカメラ起動
   - バーコード検出で自動停止し、次の処理へ分岐
   - 「手動入力」ボタン（JAN コードを直接入力するフォールバック）
2. **発注リスト（中央）**
   - カード/テーブル形式で `品名 / 数量(± で増減) / 発注先(セレクト) / 削除` を表示
   - 上部に「合計 N 件」などの件数表示
3. **操作エリア（下部）**
   - 「CSV ダウンロード」ボタン
   - 「リストをクリア」ボタン（確認ダイアログあり）
   - 「商品マスタを管理」ボタン（同一画面内で展開 or 別セクションを表示）

### 3.3 スキャン時の分岐

```
バーコード検出
├── 既知 JAN（マスタにある）
│   ├── 発注リストに同一 JAN の行あり → 数量 +1
│   └── 発注リストに行なし → マスタのデフォルト発注先で新規行を追加
└── 未知 JAN（マスタにない）
    └── モーダルを開く
        ├── 品名（必須）
        ├── 発注先（ラジオ: アスクル/カウネット/Amazon/診療、必須）
        ├── 「登録して追加」→ マスタに保存 + 発注リストに新規行追加
        └── 「キャンセル」→ 何もしない
```

## 4. データモデル

### 4.1 商品マスタ（localStorage キー: `barcode-order:products`）

```json
{
  "4901234567890": {
    "name": "アルコール綿 100包",
    "defaultSupplier": "アスクル"
  },
  "4909876543210": {
    "name": "ディスポ手袋 M 100枚",
    "defaultSupplier": "診療"
  }
}
```

- キー: JAN コード（文字列）
- 値: `{ name: string, defaultSupplier: "アスクル" | "カウネット" | "Amazon" | "診療" }`

### 4.2 発注リスト（localStorage キー: `barcode-order:order`）

```json
[
  {
    "id": "uuid-1",
    "barcode": "4901234567890",
    "name": "アルコール綿 100包",
    "quantity": 3,
    "supplier": "アスクル",
    "addedAt": "2026-04-24T10:15:00+09:00"
  }
]
```

- 配列。1 行 1 アイテム
- 同じ `barcode` を再スキャンすると該当行の `quantity` が +1
- `supplier` はマスタの `defaultSupplier` を初期値とし、リスト上で変更可能（マスタは変更しない）

## 5. 主要ロジック

- `addScannedBarcode(jan)`:
  1. マスタに存在しなければモーダル → 入力後 `registerProduct` + `addOrderLine`
  2. マスタに存在すれば、発注リストに同 JAN 行があるか確認 → あれば `quantity +=1`、なければ `addOrderLine`
- `registerProduct(jan, name, supplier)`: マスタに保存して永続化
- `addOrderLine(jan)`: マスタから品名・デフォルト発注先を引いて新規行を発注リストに追加
- `updateQuantity(id, delta)` / `updateSupplier(id, supplier)` / `removeLine(id)`: 発注リストの行操作
- `clearOrder()`: 発注リストを空に（確認ダイアログあり）
- `exportCsv()`: 発注リスト全件を CSV として生成、ダウンロード
- `renderOrder()` / `renderMaster()`: 画面再描画

## 6. CSV 仕様

- 文字コード: **UTF-8 with BOM**（Excel 日本語版の文字化け防止）
- 改行コード: `\r\n`
- ヘッダー行: `品名,数量,発注先,JANコード`
- 値に `,` `"` `\r` `\n` が含まれる場合は `"` で囲み、内部の `"` は `""` にエスケープ
- ファイル名: `order-YYYYMMDD-HHmm.csv`（例: `order-20260424-1015.csv`）
- 出力範囲: 発注リストの全行（発注先での絞り込みなし）

## 7. エラーハンドリング

| ケース | 対応 |
| --- | --- |
| カメラ許可が拒否された | エラーメッセージを表示し「手動入力」に誘導 |
| `getUserMedia` 非対応ブラウザ | カメラ機能を隠し、手動入力のみで動作 |
| `localStorage` が無効/満杯 | エラーメッセージを表示（通常は発生しない） |
| 未登録商品モーダルで品名が空 | 「登録して追加」ボタンを無効化 |
| 手動入力の JAN コードが不正 | 8 桁または 13 桁の数字のみを許容、それ以外はエラー表示 |
| 同一 JAN が連続スキャンされる | 二重登録防止のため 1.5 秒程度のデバウンスを入れる |

## 8. テスト方針

### 自動テスト（純粋ロジック）

- 対象: CSV 生成関数、マスタ操作、数量カウント、発注リスト操作
- 簡易な `test.html` を用意し、ブラウザで開くとコンソールに結果が出る形式
- フレームワーク未使用（`console.assert` 程度で十分）

### 手動テスト（実機）

- iPhone Safari / Android Chrome でのカメラ動作確認
- 以下のシナリオを受け入れ基準とする:
  1. 未登録バーコードをスキャン → モーダル表示 → 登録 → リストに追加される
  2. 同じバーコードを再スキャン → リストの数量が +1 される
  3. リスト上で数量・発注先を変更できる
  4. 行の削除・全クリアができる
  5. ページをリロードしてもデータが保持される
  6. CSV をダウンロードして Excel で開き、日本語が文字化けしない
  7. 商品マスタ管理画面で登録済み商品の編集・削除ができる
  8. カメラ拒否時に手動入力で同じ操作ができる

## 9. スコープ外（今回やらないこと）

- サーバー/クラウド連携
- 複数端末間でのデータ同期
- 発注メール送信、API 経由での発注実行
- 外部 JAN コード DB 連携（商品情報の自動取得）
- バーコード以外（QR など）からの商品情報入力
- ユーザー認証
